# Structure du Projet TP13 DDD

## Organisation des Fichiers

### `/analysis/`
Contient les analyses détaillées pour chaque phase du TP :
- `phase-1-core-domain.md` - Analyse stratégique du domaine
- `phase-2-ubiquitous-language.md` - Construction du langage métier
- `phase-3-tactical-modeling.md` - Entités, Value Objects, Agrégats, Services
- `phase-4-bounded-contexts.md` - Architecture distribuée
- `phase-5-use-cases.md` - Validation par cas d'usage
- `phase-6-synthesis.md` - Document d'architecture final

### `/diagrams/`
Schémas et diagrammes illustrant le modèle :
- `domain-overview.mmd` - Vue d'ensemble du domaine
- `aggregates.mmd` - Diagrammes des agrégats
- `context-map.mmd` - Carte des contextes bornés
- `entity-relationships.mmd` - Relations entre entités
